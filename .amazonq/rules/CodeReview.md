# CRITICAL CODE REVIEW STANDARDS

## MANDATORY REQUIREMENTS

### ⚠️ CRITICAL: Security Assessment
- **ALL code MUST be reviewed for security vulnerabilities**
- **EVERY security risk MUST be identified and addressed**
- Code with unresolved security issues will be rejected

### ⚠️ CRITICAL: Code Quality Standards
- **ALL code MUST meet industry-grade quality standards**
- **EVERY function MUST be reviewed for performance and maintainability**
- Poor quality code will NOT be accepted

### ⚠️ CRITICAL: Documentation Review
- **ALL code MUST have comprehensive comments and logging**
- **EVERY module MUST include proper documentation**
- Undocumented code will be rejected

### ⚠️ CRITICAL: Business Logic Validation
- **ALL business logic MUST be thoroughly reviewed for correctness**
- **EVERY algorithm MUST be validated for accuracy and efficiency**
- Illogical or inefficient implementations will be rejected

### ⚠️ CRITICAL: Testing Coverage
- **ALL code MUST include appropriate test coverage**
- **EVERY critical path MUST be tested**
- Code without adequate testing will be rejected

### ⚠️ CRITICAL: Best Practices Compliance
- **ALL code MUST follow established coding standards**
- **EVERY implementation MUST adhere to industry best practices**
- Non-compliant code will be rejected

---
**VIOLATION OF THESE CODE REVIEW STANDARDS WILL RESULT IN CODE REJECTION**
